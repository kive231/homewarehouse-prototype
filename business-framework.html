定位是个人/家庭物品管理 App，不做企业 WMS。第一版围绕「找得到、放得清、扫得出、用得掉、补得上、查得到」设计。

01. 项目定位与边界

结论：先做「家庭物品库存 + 空间定位 + 分类维护 + 扫码追溯」四件事。

核心价值

帮用户知道东西在哪里、还有多少、什么时候该处理。

核心场景

录入物品、绑定位置、扫码查看、消耗补货、移动位置、查看记录。

不做什么

不做采购审批、供应商、财务、复杂权限、企业多仓 ERP。

产品体验按手机系统文件管理器设计：一级页进入管理，管理页进入空间/分类，每次只看当前层级，返回上一层，不展示复杂树。

存货分类管理：大类 → 子类 → 细类

3. 管理 └─ 9. 存货分类维护（大类列表） ├─ 9-2. 子类列表 │ └─ 9-3. 细类列表 └─ 10. 新增/编辑分类

分类严格三层，不做标签池、模板关联、无限树。空间也是三层主导航，盒子内部可后端支持嵌套，但第一版 UI 不暴露复杂层级。

03. 核心业务流程

3.1 建仓流程

1
创建空间结构
管理 → 空间结构维护 → 新增柜子 → 新增层 → 新增盒子。
2
创建存货分类
管理 → 存货分类维护 → 新增大类 → 子类 → 细类。
3
录入物品
首页/扫码页进入新增物品，选择位置节点和细类，填写数量、单位、保质期。
4
绑定二维码
给盒子或物品生成二维码，扫码后直接进入详情或盒子清单。

3.2 找物流程

1
搜索找物
首页搜索物品名、分类、位置编号。
2
浏览找物
按柜子 → 层 → 盒子逐级进入，符合手机文件夹操作习惯。
3
扫码找物
扫盒子码展示盒内物品；扫物品码展示物品详情。

3.3 库存变动流程

消耗物品详情 → 消耗 → 输入数量 → 扣减库存 → 生成消耗记录 → 低库存判断。
补货物品详情 → 补货 → 输入数量/批次/保质期 → 增加库存 → 生成补货记录。
移动物品详情 → 移动位置 → 选择新柜子/层/盒子 → 保存 → 生成移动记录。
归档物品详情 → 归档 → 不再参与首页库存统计，但保留历史记录。

04. 领域模型

对象职责关键字段设计约束
User账户id, nickname, avatar, status第一版个人账户，预留 owner_id。
LocationNode空间节点id, parent_id, type, name, code, sorttype 只暴露 CABINET/LAYER/BOX 三层。
CategoryNode存货分类id, parent_id, level, name, icon, colorlevel 固定 1/2/3，不做标签。
Item物品主档name, category_id, location_id, unit, qty必须绑定一个位置和一个分类。
InventoryRecord库存流水type, before_qty, change_qty, after_qty所有消耗、补货、移动都写记录。
QRCode扫码入口target_type, target_id, code_url可绑定盒子或物品。

06. 技术框架

App

Flutter。先做 Web 预览对齐 UI，再由 GitHub Actions 打 APK。

后端

Spring Boot 3 + Spring Security + JWT/Refresh Token。

数据库

PostgreSQL 优先；MySQL 可替代。核心靠邻接表建树。

部署

Docker Compose：app-api、postgres、nginx，可后置 Redis/MinIO。

权限

第一版个人账户。后续扩展 Household、成员、私人物品。

文件

二维码图片可动态生成；头像/图片后置对象存储。

07. 后端 API 边界

模块接口范围备注
Auth/auth/login, /auth/refresh, /auth/logout前期不开放注册。
LocationCRUD、按父级查询、移动排序、二维码绑定支撑柜子/层/盒子。
CategoryCRUD、按父级查询、移动排序严格校验最大 3 层。
Item新增、编辑、详情、搜索、归档查询需要支持位置、分类、关键词。
Inventoryconsume、replenish、move、records所有变动走服务层事务。
QRCodegenerate、scan、bind、unbindscan 返回 target_type 和 target_id。

08. MVP 排期

1
第 1 阶段：基础框架
登录、首页、底部导航、管理页、数据库模型。
2
第 2 阶段：空间与分类
空间三层导航、分类三层导航、增删改查。
3
第 3 阶段：物品与库存
新增物品、详情、搜索、消耗、补货、移动。
4
第 4 阶段：扫码与记录
二维码生成/扫描、库存流水、首页临期/低库存提示。

验收标准：用户不需要学习树结构，能像手机文件夹一样逐层进入空间和分类；每个物品都能通过搜索、分类、位置、扫码四种方式找到。
