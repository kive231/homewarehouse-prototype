## 2. Overlap — 冗余识别

### 冗余组 A：空间树 (4→5→7→8→6) — 5 个页面

页面 4、5、7、8 是一条树状链：空间列表 → 柜子 → 层 → 盒子。每一级都是一个独立页面，结构完全相同（列表 + 概览 + 操作按钮），只是节点层级不同。页面 6 是这四级的编辑表单。

**本质：1 个树状浏览功能被拆成 5 个页面。**

### 冗余组 B：分类树 (9→9-2→9-3→10) — 4 个页面

同冗余组 A。大类 → 子类 → 细类 → 表单，结构完全镜像空间树。

**本质：分类层级和空间层级是两个维度，但 UI 模式完全相同。**

### 冗余组 C：物品操作 (16/18/19/20) — 4 个页面

消耗、补货、移动都是对单个物品的数量/属性变更操作。页面 18/19 的内容几乎一样（物品卡片 + 数量 stepper + 确认按钮），20 是多了一个目标选择。这些操作的触发点都在物品详情页(16)。

**本质：物品详情页上的 3 个按钮，被拆成 3 个独立页面。**

### 冗余组 D：我的页面散落 — 6 个页面

个人资料(25)、用户偏好(26)、二维码管理(11)、生成二维码(12)、导入导出(27)、关于帮助(28) 在原型中排列为 6 个独立"shot"。其中 25 和 26 都是设置项，11 和 12 是母子关系。

**本质：6 个设置项散落，没有统一的设置中心。**

### 冗余组 E：扫码 (23) vs 新增物品 (17)

扫码的原始意图是"识别物品码→跳转物品详情/空间详情"，新增物品是"录入手动输入"。两者是不同的入口，功能不冗余。但当前原型中扫码页面只是一个扫码框，缺少"扫到后"的后续承接。

**本质：不冗余，但扫码的后续流程缺失展示。**

## 4. Uncover — 暴露缺失

非冗余问题，而是流程中确实缺的：

1. **扫码后续流程缺失。** 页面 23 (扫码) 扫完后，物品码 → 物品详情，位置码 → 空间详情。但原型中没有展示这个跳转衔接（比如扫码结果 overlay）。
2. **物品详情页缺少"操作记录"。** 当前只有信息展示和操作按钮。用户消耗/补货后，看不到该物品的历史流水。
3. **首页缺少个性化 widget。** 首页(2) 目前是固定内容。如果用户想自定义显示的统计数据（比如只看"画材"分类的临期提醒），没有入口。
4. **缺少批量操作。** 所有操作（消耗、补货、移动、贴码）都是单物品操作。实际场景中用户可能一次处理多个物品。

## 5. Schema — 优化后信息架构

```
底部导航 (4 Tab):

├── Tab1: 首页
│   └── 仪表盘 (概览卡片 + 快捷操作 + 最近使用 + 全局搜索)

├── Tab2: 物品
│   ├── 全部物品 (分类筛选标签 + 搜索 + 列表)
│   ├── 物品详情 (信息 + 消耗/补货/移动按钮 → 弹出操作层 + 操作历史)
│   └── 新增物品 (表单 + 扫码绑定)

├── Tab3: 空间与分类
│   ├── 空间浏览 (树状一体: 空间→柜子→层→盒子, 点击下钻 + 面包屑返回)
│   └── 分类浏览 (树状一体: 大类→子类→细类, 点击下钻 + 面包屑返回)

├── Tab4: 我的
│   ├── 个人中心 (头像 + 数据统计)
│   ├── 二维码管理 (列表 + 浮动生成按钮 → 弹出表单)
│   ├── 出入库记录 (流水列表, 按消耗/补货/移动筛选)
│   ├── 扫码 (独立扫码入口, 扫物品码→详情, 扫位置码→空间)
│   └── 设置 (个人资料 / 偏好 / 导入导出 / 关于)
```

**最终页面数: 13 个** (Tab1:1, Tab2:3, Tab3:2, Tab4:5 + 扫码独立 + 登录独立)

## 核心用户路径 (最短操作步数)

```
登录 → 首页:  登录(1) → 首页(1步)

扫码看物品:  首页点扫码 → 扫物品码 → 物品详情 (3步)
扫码看位置:  首页点扫码 → 扫位置码 → 空间详情 (3步)

查找物品:  首页搜索 → 点结果 → 物品详情 (3步)

消耗物品:  全部物品 → 点物品 → 物品详情 → 点"消耗" → 确认数量 (5步)
补货物品:  同上 (5步)
移动物品:  全部物品 → 点物品 → 物品详情 → 点"移动" → 选目标 → 确认 (6步)

新增物品:  首页"新增物品" → 填表 → 保存 (3步)
新增空间:  空间浏览 → 点"+" → 弹出表单 → 保存 (4步)
新增分类:  分类浏览 → 点"+" → 弹出表单 → 保存 (4步)

查看记录:  我的 → 出入库记录 (2步)
生成二维码: 我的 → 二维码管理 → 点"生成" → 弹出表单 → 保存 (5步)
```

## 总结

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 总页面数 | 28 | 13 |
| 空间相关页面 | 5 | 1 (树状一体) |
| 分类相关页面 | 4 | 1 (树状一体) |
| 物品操作页面 | 4 | 0 (并入详情页弹窗) |
| 设置相关页面 | 6 | 1 (统一设置页) |
| 底部导航 Tab | 3 (首页/管理/物品/我的) | 4 (首页/物品/空间与分类/我的) |

> **关键建议**: 保留原版薄荷绿配色和设计风格，只重构页面结构。UI 层不动，动信息架构层。
